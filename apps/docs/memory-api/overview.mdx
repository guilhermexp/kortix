## Authentication

Faça login no painel self-hosted (`http://localhost:3000`) com um usuário administrador e gere um token em **Configurações → API Keys**. Todas as requisições devem enviar o header `Authorization: Bearer <token>`.

All API requests require authentication using an API key. Include your API key as follows:

<CodeGroup>

```bash cURL
Authorization: Bearer YOUR_API_KEY
```

```bash Node.js
const API_URL = process.env.SUPERMEMORY_API_URL ?? "http://localhost:4000"
const API_KEY = process.env.SUPERMEMORY_API_KEY

const response = await fetch(`${API_URL}/v3/documents`, {
  headers: { Authorization: `Bearer ${API_KEY}` },
})
```

```python Python
import os

API_URL = os.getenv("SUPERMEMORY_API_URL", "http://localhost:4000")
API_KEY = os.environ["SUPERMEMORY_API_KEY"]
```

</CodeGroup>

## Clientes e SDKs

O backend expõe uma API REST em `http://localhost:4000/v3`. Use qualquer cliente HTTP (fetch, axios, requests, curl). Os pacotes `@repo/lib` e `@supermemory/tools` dentro do monorepo já encapsulam chamadas para o frontend, mas não há dependência externa obrigatória.

## Add your first memory

<CodeGroup>

```bash cURL
curl "$SUPERMEMORY_API_URL/v3/documents" \
  --request POST \
  --header 'Content-Type: application/json' \
  --header "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  --data '{"name":"Primeira memória","type":"text","projectId":"default","content":"Este é o conteúdo da minha primeira memória."}'
```

```typescript Node.js
await fetch(`${API_URL}/v3/documents`, {
  method: "POST",
  headers: {
    Authorization: `Bearer ${API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Primeira memória",
    type: "text",
    projectId: "default",
    content: "Este é o conteúdo da minha primeira memória.",
  }),
})
```

```python Python
requests.post(
    f"{API_URL}/v3/documents",
    headers={
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json",
    },
    json={
        "name": "Primeira memória",
        "type": "text",
        "projectId": "default",
        "content": "Este é o conteúdo da minha primeira memória.",
    },
    timeout=30,
)
```

</CodeGroup>

This will add a new memory to your supermemory account.

Try it out in the [API Playground](/api-reference/manage-memories/add-memory).

## Content Processing

<Accordion title="Processing steps" icon="sparkles">
  When you add content to supermemory, it goes through several processing steps:

    1. **Queued**: Initial state when content is submitted
    2. **Extracting**: Content is being extracted from the source
    3. **Chunking**: Content is being split into semantic chunks
    4. **Embedding**: Generating vector embeddings for search
    5. **Indexing**: Adding content to the search index
    6. **Done**: Processing complete
</Accordion>

<Accordion title="Advanced Chunking" icon="sparkles">
  The system uses advanced NLP techniques for optimal chunking:

    - Sentence-level splitting for natural boundaries
    - Context preservation with overlapping chunks
    - Smart handling of long content
    - Semantic coherence optimization

</Accordion>

## Search your memories

<CodeGroup>

```bash cURL
curl "$SUPERMEMORY_API_URL/v3/search" \
  --request POST \
  --header 'Content-Type: application/json' \
  --header "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  --data '{"q": "Este é o conteúdo da minha primeira memória.", "limit": 3}'
```

```typescript Node.js
await fetch(`${API_URL}/v3/search`, {
  method: "POST",
  headers: {
    Authorization: `Bearer ${API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ q: "Este é o conteúdo da minha primeira memória.", limit: 3 }),
})
```

```python Python
requests.post(
    f"{API_URL}/v3/search",
    headers={
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json",
    },
    json={"q": "Este é o conteúdo da minha primeira memória.", "limit": 3},
    timeout=30,
)
```

</CodeGroup>

Try it out in the [API Playground](/api-reference/search-memories/search-memories).
